# Productization Foundation: npm, Docs, Landing Page

> **Goal:** Make cuekit installable and discoverable before any marketing or demos.

## Problem

- `@cuekit/cli` is `private: true` — not published to npm
- Installation requires `bun install -g github:takemo101/cuekit` — only Bun users, no npm/yarn/pnpm
- No landing page — first-time visitors cannot understand what cuekit is in 30 seconds
- Docs are design-heavy, tutorial-light — new users cannot follow "how to start"
- README is long but not structured as "sales copy"

## Required Deliverables

### 1. npm Publication

#### Current State

```json
// packages/cli/package.json
{
  "name": "@cuekit/cli",
  "version": "0.0.11",
  "private": true,
  "type": "module",
  "exports": { ".": "./src/index.ts" },
  "bin": { "cuekit": "./src/bin.ts" }
}
```

Problems:
- `private: true` → npm refuses publish
- `exports` points to `./src/index.ts` — TypeScript source, not bundled JS
- `bin` points to `./src/bin.ts` — same problem
- Dependencies use `workspace:*` — npm cannot resolve these

#### Required Changes

**Step 1: Bundle generation**

`bin/cuekit.js` already exists as a bundled entry point (generated by some build step). Confirm this is the publish artifact.

```json
// packages/cli/package.json (for publish)
{
  "name": "@cuekit/cli",
  "version": "0.0.11",
  "private": false,
  "type": "module",
  "main": "./dist/index.js",
  "bin": { "cuekit": "./dist/bin.js" },
  "files": ["dist/", "README.md", "LICENSE"],
  "engines": { "node": ">=20" },
  "dependencies": {
    // Bundled — no runtime deps if using single-file bundle
  }
}
```

**Step 2: Publish flow**

```bash
# 1. Build bundle
cd packages/cli && bun run build

# 2. Copy bundle to dist/
cp bin/cuekit.js dist/bin.js

# 3. Publish (workspace deps are already bundled)
cd packages/cli && npm publish --access public
```

**Step 3: Global install**

```bash
npm install -g @cuekit/cli
# or
yarn global add @cuekit/cli
# or
pnpm add -g @cuekit/cli
# or
bun install -g @cuekit/cli
```

#### Verification

```bash
which cuekit
cuekit --version
cuekit doctor
cuekit team list
```

1. Remove `private: true` from `@cuekit/cli`
2. Verify `bin/cuekit.js` bundle works standalone
3. Add `files`, `engines`, `publishConfig` to `packages/cli/package.json`
4. Test `npm pack` and install from tarball

### Week 3: Landing Page

2. Add asciinema placeholder
3. Deploy to GitHub Pages
4. Link from README and repo description
